Auflistung der Klassen
Hier folgt die Aufzählung aller Klassen, Strukturen, Varianten und Schnittstellen mit einer Kurzbeschreibung:
JADE.BearbeitenFensterForm-Objekt(Windows Fenster), in dem das Trennen eines Token in zwei neue Token realisiert wird
JADE.DatenEine Klasse, die für das Speichern der Arraylist mit unseren Daten für den eingegebenen Text (die einzelnen Sätze und Token) gedacht ist. Sie wurde auch für den Zweck angelegt, spätere Änderungen wie das Speichern von Zusatzinformationen wie Übersetzungsmöglichkeiten zu erleichtern
JADE.HauptFensterForm-Objekt(Windows Fenster) für das Hauptfenster unserer Anwendung
JADE.SaveObjektEine Klasse, die als Container für zu speichernde Daten fungiert. Sie wird um Datenstruktur zum Abspeichern erweitert werden, kann aber potentiell auch andere Daten zum Speichern erhalten
JADE.SaverKlasse welche zum Speichern unserer Datenstruktur verwendet wird. Es wurde auch hier Rücksicht auf Erweiterbarkeit genommen. Zum Speichern wird später ein String für den Pfad und ein SaveObjekt benötigt
JADE.SearchEngineDie Klasse SearchEngine enthält die Funktionen zum Suchen eines Tokens im Wörterbuch. Mit Hilfe von Linq to XML wird wadoku.xml durchsucht, um für ein gesuchtes Token entsprechende Einträge zu finden. Dabei gibt es die Möglichkeit, nach genau übereinstimmenden Einträgen zu suchen oder aber nach allen Einträgen, die wie das Token beginnen. So können eventuelle falsche Zerlegungen des TinySegmenters mit Hilfe von Wadoku und der Bearbeiten-Funktion verbessert werden
JADE.SegmenterSegmenter-Klasse enthält die Funktionen zum Tokenisieren des Eingabe Textes
